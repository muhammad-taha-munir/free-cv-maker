Project History

Initial State

The project began as a functional skeleton of a CV Maker application. Key features included:

  • Basic React structure with Vite
  • JSON Import/Export functionality for CV data
  • A basic editor and preview panel
  • A non-functional "Export PDF" button using react-to-print

Enhancement Phase Goals

The primary objective was to transform the skeleton into a feature-rich application by:

  • Implementing a Design Control Panel for switching templates and fonts
  • Creating 3 Professional Templates (Modern, Classic, Creative)
  • Fixing the PDF Export functionality to ensure direct downloads
  • Polishing the UI for a professional aesthetic

UX Fix: PDF Export (Print Dialog)

2025-11-30

Problem Statement

The "Export PDF" button used react-to-print, which only opened the browser's print dialog. The user expected a direct file download and reported the dialog as "not visible" or "hanging".

Obstruction / Original Code

const handlePrint = useReactToPrint({
    content: () => componentRef.current,
    documentTitle: 'My CV',
});

Solution Steps

  • Removed react-to-print
  • Switched to html2pdf.js (and later direct html2canvas + jspdf) to force a file download

Critical Bug Fix: PDF Export Hang & Error

2025-11-30

Problem Statement

1. The UI would freeze (hang) during PDF generation.
2. Generation failed with error: Attempting to parse an unsupported color function "oklch".

Obstruction / Root Cause

  • Hang: Synchronous heavy calculation on the main thread
  • Error: Tailwind CSS v4 defaults to oklch color space, which html2canvas does not support

Solution Steps

  • Implemented isGenerating state to show a loading spinner and disable the button
  • Replaced html2pdf.js wrapper with direct html2canvas + jspdf implementation for better control
  • Crucial Fix: Overrode Tailwind's default colors in index.css with standard HEX values

Corrected Code (Color Override)

@theme {
  /* Override colors with HEX for html2canvas compatibility */
  --color-gray-50: #f9fafb;
  --color-gray-100: #f3f4f6;
  /* ... other colors ... */
  --color-blue-600: #2563eb;
}

Corrected Code (Export Logic)

const handlePrint = async () => {
    setIsGenerating(true);
    try {
        const html2canvas = (await import('html2canvas')).default;
        const { jsPDF } = await import('jspdf');
        
        const canvas = await html2canvas(element, { scale: 2, useCORS: true, backgroundColor: '#ffffff' });
        const imgData = canvas.toDataURL('image/jpeg', 0.95);
        
        const pdf = new jsPDF('p', 'mm', 'a4');
        pdf.addImage(imgData, 'JPEG', 0, 0, pdfWidth, finalHeight);
        pdf.save('my-cv.pdf');
    } catch (err) {
        alert(`Failed to generate PDF: ${err.message}`);
    } finally {
        setIsGenerating(false);
    }
};

Bug Fix: Skills Layout in Classic Template

2025-12-01 01:15

Problem Statement

The horizontal layout and underlining features for the "Skills" section were initially implemented only for the Modern Template. The Classic Template still displayed skills vertically with extra fields.

Solution Steps

  • Applied the same logic from ModernTemplate to ClassicTemplate
  • Used flex-wrap and justify-center to align skills horizontally in the center, matching the Classic theme's centered aesthetic
  • Implemented the underline toggle check
  • Note: The CreativeTemplate was intentionally left unchanged as the vertical layout suits its design better

Quality Fix: PDF Export & Spacing

2025-12-01 01:30

Problem Statement

1. PDF export quality was low (blurry text).
2. Underlines in "Skills" and Section Headers were too close to the text, sometimes merging with it.

Solution Steps

  • PDF Quality: Increased html2canvas scale from 2 to 4
  • Spacing: Increased padding-bottom for Skills items and Section Headers in both Modern and Classic templates
  • Added leading-relaxed to skills to improve line height

Code Snippet (App.jsx)

const canvas = await html2canvas(element, {
    scale: 4, // Increased from 2 for high quality
    useCORS: true,
    logging: false,
    backgroundColor: '#ffffff'
